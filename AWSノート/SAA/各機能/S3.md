# 🗂 Amazon S3 解説

## 1. 基本概要

- **オブジェクトストレージサービス**  
    → ファイルを「オブジェクト」として保存する仕組み
    
- **スケーラブルで耐久性の高いストレージ**  
    → 公式耐久性は **99.999999999%（11ナインズ）**

- 無制限に保存可能（容量制限なし）

- 代表的な利用用途：
    
    - バックアップ / アーカイブ
        
    - 静的Webサイトホスティング
        
    - データレイク（分析基盤のデータ保存先）
        
    - 画像 / 動画 / アプリ配布ファイルの配信
        

---

## 2. アーキテクチャ

- **バケット（Bucket）**
    
    - S3内のトップレベルのコンテナ（ストレージのフォルダのような概念）
        
    - バケット名はグローバルで一意
        
- **オブジェクト（Object）**
    
    - 実際に保存するファイル（例: `image.jpg`, `data.csv`）
        
    - 各オブジェクトは「キー（パスのようなもの）」と「バリュー（データ本体）」から構成
        

---

## 3. 特徴

- **高耐久性**
    
    - 複数のAZに自動で冗長保存
        
- **高可用性**
    
    - リージョン内のどこからでもアクセス可能
        
- **拡張性**
    
    - 無制限の容量をスケーラブルに利用可能
        
- **柔軟なセキュリティ**
    
    - IAMポリシー、バケットポリシー、ACL、暗号化対応
        
- **多様なアクセス制御**
    
    - プライベートアクセス / パブリック公開 / 署名付きURL
        

---

## 4. ストレージクラス（コストとアクセス頻度で使い分け）

- **S3 Standard**
    
    - 頻繁にアクセスするデータ向け
        
- **S3 Intelligent-Tiering**
    
    - アクセス頻度に応じて自動で最適化
        
- **S3 Standard-IA（Infrequent Access）**
    
    - 低頻度アクセスデータ向け（コスト削減）
        
- **S3 One Zone-IA**
    
    - 単一AZ保存（コスト最安だが可用性低い）
        
- **S3 Glacier / Glacier Deep Archive**
    
    - 長期アーカイブ用（取り出しに数分～数時間かかるが格安）
        

---

## 5. セキュリティ

- **暗号化**
    
    - SSE-S3（AWS管理キー）
        
    - SSE-KMS（KMSカスタマーマスターキー）
        
    - SSE-C（ユーザー提供キー）
        
- **アクセス制御**
    
    - IAMポリシー / バケットポリシー / ACL
        
    - Block Public Access設定で誤公開を防止
        

---

## 6. 代表的なユースケース

- **静的Webサイトホスティング**  
    → HTML / CSS / JSをS3に置き、CloudFront経由で配信
    
- **バックアップストレージ**  
    → DBスナップショットやログファイルの保存先
    
- **データレイク**  
    → Athena, Redshift Spectrum, EMR などの分析基盤のデータ置き場
    
- **コンテンツ配信**  
    → CloudFront + S3でグローバルCDN配信
    

---

## 7. 他サービスとの連携

- **CloudFront** → グローバル配信（CDN）
    
- **Athena** → SQLで直接S3データを分析
    
- **Glue** → データカタログ化
    
- **Redshift Spectrum** → S3にあるデータを直接分析
    
- **Backup** → バックアップ対象として利用
    

---

# ✅ まとめ

Amazon S3は、

- **無制限・高耐久のオブジェクトストレージ**
    
- **用途に応じたストレージクラスでコスト最適化**
    
- **IAMや暗号化で強力なセキュリティ**
    
- **CloudFrontやAthenaなどAWSサービスと親和性が高い**
