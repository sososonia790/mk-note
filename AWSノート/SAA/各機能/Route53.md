# 🌐 Amazon Route 53 概要

## 1. Route 53とは？

- AWSが提供する **スケーラブルで高可用なDNSサービス**
    
- 名前の由来：「Port 53」(DNSで利用されるポート番号)
    
- 主な役割：
    
    - **ドメイン名登録（Registrar機能）**
        
    - **DNSルーティング（名前解決）**
        
    - **ヘルスチェック（可用性監視）**
        

---

## 2. 主な機能

### 🔹 (1) ドメイン登録

- Route 53を使って独自ドメインを購入・管理可能
    
- 自動更新も可能
    
- 他社レジストラで取得したドメインもRoute 53に移管可能
    

---

### 🔹 (2) ホストゾーン

- **DNSのレコード管理単位**
    
- 種類：
    
    - **パブリックホストゾーン** → インターネット公開用（Webサイトなど）
        
    - **プライベートホストゾーン** → VPC内部専用のDNS解決用
        

---

### 🔹 (3) レコードタイプ（代表例）

- **Aレコード** → ドメイン名をIPv4アドレスへ変換
    
- **AAAAレコード** → IPv6アドレスへ変換
    
- **CNAMEレコード** → 別のドメイン名にエイリアス
    
- **MXレコード** → メールサーバー指定
    
- **SOAレコード** → ゾーンの権威情報（キャッシュ更新間隔など）
    

✅ 試験ポイント：  
Route 53は **Aliasレコード**をサポートし、S3・CloudFront・ALBなどAWSリソースを直接指定できる（通常のDNSにはないAWS特有の機能）。

---

### 🔹 (4) ルーティングポリシー

- **Simple Routing** → 単一リソースにルーティング
    
- **Weighted Routing** → 複数リソースに比率を割り当て
    
- **Latency-based Routing** → ユーザーに最もレイテンシが低いリージョンへ
    
- **Failover Routing** → プライマリ/セカンダリの冗長構成
    
- **Geolocation Routing** → ユーザーの地理的位置に基づきルーティング
    
- **Multi-Value Answer Routing** → 複数の正常なリソースを返す（簡易ロードバランシング）
    

---

### 🔹 (5) ヘルスチェック

- Route 53が外部からエンドポイントの稼働状況を監視
    
- 異常検知時 → 別の正常リソースにフェイルオーバー可能
    
- CloudWatchとも連携して柔軟な監視が可能
    

---

## 3. 代表的なアーキテクチャ利用例

1. **S3静的Webサイト + Route 53**
    
    - AliasレコードでS3バケットをドメインに直接割り当て
        
2. **ALB + Route 53**
    
    - Route 53でドメインを管理
        
    - ALBをAliasレコードとして登録
        
    - 世界規模ではLatency-based Routingで最適リージョンへ
        
3. **マルチリージョン冗長構成**
    
    - US-East-1をプライマリ、AP-Northeast-1をセカンダリ
        
    - Route 53のFailover Routingを設定
        
    - ヘルスチェック異常で自動的にセカンダリへ切替
        

---

## 4. 試験対策ポイント（SAA / SAP）

- **CLFレベル**
    
    - Route 53は「DNSサービス」だと理解すればOK
        
- **SAAレベル**
    
    - レコードタイプ（A, CNAME, Alias）
        
    - ルーティングポリシー（特にWeighted, Latency, Failover）
        
    - パブリック vs プライベートホストゾーンの違い
        
- **SAPレベル**
    
    - マルチリージョン設計（Route 53 + CloudFront + ALBの組み合わせ）
        
    - Route 53ヘルスチェックの詳細挙動
        
    - グローバル災害対策でのRoute 53フェイルオーバー設計
        

---

# ✅ まとめ

- **Route 53はAWSのDNSサービス**
    
- ドメイン登録、名前解決、ヘルスチェックを提供
    
- **Aliasレコード**によりAWSリソースを直接参照できるのが強み
    
- **ルーティングポリシー**（Weighted / Latency / Failover）が設計の肝
    
- **マルチリージョン冗長構成**ではRoute 53のフェイルオーバーが重要
